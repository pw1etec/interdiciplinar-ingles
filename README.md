
### 1. Markdown: UrbanFlow (Mobilidade Urbana)


# 🚍 UrbanFlow - Smart Mobility App

## 🏢 About the Company
**UrbanFlow** is a European startup focused on smart mobility. Our mission is to simplify public transportation and commuting in large cities, making travel for international tourists and students more sustainable and stress-free.

## 🚨 Problem Statement
Tourists waste too much time trying to understand local traffic and transit systems. Traditional routing apps have excessive text in local languages, visual pollution, and do not clearly show where to find integrated services (like bike rentals or subway stations). The result is anxiety and inefficient routing.

## 🎯 Technical & UI/UX Requirements
* **Universal Iconography:** The design must prioritize intuitive icons over blocks of text.
* **Accessibility:** High contrast for outdoor use (sunlight) and support for *Dark/Light Mode*.
* **Fluid Navigation:** Easy-to-reach menus for one-handed use (Mobile First approach).

## 📋 Product Backlog (User Stories)
* [ ] **US01:** As a tourist, I want to view a clean map on the home screen, so that I can quickly locate my position.
* [ ] **US02:** As a user, I want to switch the map view between "Subway", "Bus", and "Bicycles" with a single click, so that I can choose my transportation method.
* [ ] **US03:** As a tourist, I want to input my origin and destination and receive three route options ("Fastest", "Cheapest", "Eco-friendly"), so that I can decide the best strategy.
* [ ] **US04:** As a user, I want to see visual alerts without needing to read complex text if a route is blocked (e.g., a roadwork icon), so that I can avoid delays.

## 📦 Deliverables
1. **Prototype (Figma):** High-Fidelity Screens (Home/Map, Routing, Transportation Options).
2. **Documentation:** Presentation script (Pitch) in English.

### 2. Markdown: TasteMatch (Gastronomia e Cultura)


# 🍣 TasteMatch - Cultural Food Platform

## 🏢 About the Company
**TasteMatch** is a global food tourism network based in Canada. We connect exchange students and travelers to the true essence of local cuisine, helping them avoid tourist traps.

## 🚨 Problem Statement
Finding authentic and affordable food in a foreign country is difficult. Furthermore, the language barrier on local menus represents a real and constant danger for users with food allergies or dietary restrictions (such as gluten, peanut, or lactose intolerance).

## 🎯 Technical & UI/UX Requirements
* **Visual Appeal:** Absolute focus on high-quality images of the dishes (the design must be mouth-watering).
* **Trust:** Use of visual badges (e.g., "Local Choice" / "Authentic").
* **Safety:** The allergy system must be the clearest and most prominent feature of the interface.

## 📋 Product Backlog (User Stories)
* [ ] **US01:** As a user, I want to see a discovery *Feed* based on my location, so that I can find the highest-rated dishes recommended by locals.
* [ ] **US02:** As a traveler, I want to access a dish's detail page to see its history, ingredients, and where it is served.
* [ ] **US03:** As a user with dietary restrictions, I want to set up my profile with my food allergies, so that the app filters recommendations safely.
* [ ] **US04:** As an allergic user, I want to see a clear, red visual alert on the dish screen if it contains any dangerous ingredients for me, so that I can eat safely.

## 📦 Deliverables
1. **Prototype (Figma):** High-Fidelity Screens (Discovery Feed, Dish Details, Profile/Allergy Setup).
2. **Documentation:** Presentation script (Pitch) in English.

### 3. Markdown: StudySync (Educação e Organização)


# 📚 StudySync - Educational Dashboard

## 🏢 About the Company
**StudySync** is an American EdTech (Educational Technology) company. We create digital tools to ensure the academic success and mental well-being of international students living far from home.

## 🚨 Problem Statement
Exchange students deal with chaotic routines: assignment deadlines, time zone clashes when trying to call family, and the difficulty of finding classmates in the same school to form study groups, which leads to isolation.

## 🎯 Technical & UI/UX Requirements
* **Visual Organization:** The Dashboard must be clean, using *Cards* to divide information without overwhelming the user (information architecture).
* **Grid/Layout:** Efficient use of screen space to accommodate a calendar, tasks, and contacts simultaneously (Desktop/Tablet First).
* **Interaction:** Must convey a sense of community and productivity.

## 📋 Product Backlog (User Stories)
* [ ] **US01:** As a student, I want to view a main Dashboard with a summary of my day (upcoming classes and urgent tasks), so that I can organize my time.
* [ ] **US02:** As an exchange student, I want a customizable world clock *widget*, so that I quickly know what time it is in my home country.
* [ ] **US03:** As a student, I want a task management tool formatted as a *Kanban* board (To Do, Doing, Done), so that I can track my assignments.
* [ ] **US04:** As a freshman, I want to use the "Academic Match" feature by entering the topics I am studying, so that the system suggests classmates on campus interested in studying together.

## 📦 Deliverables
1. **Prototype (Figma):** High-Fidelity Screens (Main Dashboard, Kanban Organizer, Match/Community Screen).
2. **Documentation:** Presentation script (Pitch) in English.

### 4. Markdown: EcoTech Exchange (Sustentabilidade)


# ♻️ EcoTech Exchange - Secure Marketplace

## 🏢 About the Company
**EcoTech Exchange** is a sustainable technology NGO based in Australia. We work to reduce global e-waste by creating safe environments for the recirculation of gadgets among young people.

## 🚨 Problem Statement
Students buy electronics abroad and, when it's time to return home, they generate e-waste. At the same time, students who need cheaper devices are afraid of being scammed on standard internet marketplaces. There is a lack of a reliable and exclusive environment for this audience.

## 🎯 Technical & UI/UX Requirements
* **Trust & Safety:** The design must convey institutional security (colors, padlocks, verification badges).
* **Clear Categorization:** Quick visual filters and badges to identify the product's condition (New, Used, Defective).
* **Usability:** Frictionless buying and selling flow, with direct communication.

## 📋 Product Backlog (User Stories)
* [ ] **US01:** As a student, I want to log in using a student ID verification or institutional email, so that I can ensure only real students access the platform.
* [ ] **US02:** As a buyer, I want to view a product catalog where each item has a clear condition rating (e.g., Grade A, B, or C), so that I avoid unpleasant surprises.
* [ ] **US03:** As a seller, I want a simple form to register my used electronics, adding photos and a price quickly.
* [ ] **US04:** As a verified user, I want to access an internal chat to negotiate with another student and arrange the exchange in a safe campus location.

## 📦 Deliverables
1. **Prototype (Figma):** High-Fidelity Screens (Login/Authentication, Catalog/Showcase, Product Screen, Negotiation Chat).
2. **Documentation:** Presentation script (Pitch) in English.
